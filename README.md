# Skinner (subscriptionsCheckApp)

Telegram-бот для автоматизации проведения розыгрышей и конкурсов. Бот проверяет подписку пользователей на заданные Telegram-каналы и сообщества ВКонтакте (VK), управляет процессом регистрации участников, ведет их учет в базе данных, а также выполняет периодическую фоновую проверку актуальности подписок. Если пользователь отписывается от обязательного ресурса — бот автоматически исключает его из участия в конкурсе.

---

## 🚀 Основные возможности

* **Интерактивное управление:** Полноценное взаимодействие через текстовые команды меню и кликабельные `Inline`-кнопки для привязки аккаунтов и отправки проверок.
* **Привязка VK-аккаунта:** Поддержка гибких форматов ввода ссылки (полный URL-адрес, короткое имя страницы или формат `@username`). Бот автоматически делает запрос к VK API (`utils.resolveScreenName`) для преобразования текстового имени в постоянный числовой ID.
* **Комплексная проверка подписок:**
  * Проверка членства в целевых Telegram-каналах через встроенный клиент `DefaultAbsSender` (Telegram Bots API).
  * Проверка подписки на указанные паблики/группы ВКонтакте с помощью реактивного WebClient (`groups.isMember`).
* **Фоновый мониторинг (Scheduler):** Реализована периодическая задача, которая каждые 45 секунд сканирует базу данных активных участников. При обнаружении факта отписки от каналов или групп VK, статус участия аннулируется.
* **Автоматическое одобрение заявок (Join Request):** Бот умеет перехватывать входящие заявки на вступление в каналы (`ChatJoinRequest`) и автоматически одобряет их через настраиваемый промежуток времени (по умолчанию — 30 секунд).
* **Административный доступ:** Наличие защищенной команды `/contest_users` для выгрузки актуального списка верифицированных участников конкурса (ID, Telegram Username, VK ID).

## 🛠 Технологический стек

* **Язык разработки:** Java 17
* **Фреймворк:** Spring Boot 3.5.5
  * **Spring Data JPA** — для работы с репозиториями пользователей.
  * **Spring WebFlux (WebClient)** — для реактивного выполнения запросов к внешнему API ВКонтакте.
  * **Spring Scheduling** — для автоматизации фоновых проверок.
* **База данных:** PostgreSQL
* **Управление миграциями:** Liquibase
* **Библиотека Telegram API:** `org.telegram:telegrambots` (версия 6.9.7.1)
* **Дополнительные утилиты:** Lombok (для сокращения шаблонного кода), Maven Wrapper

## 📂 Структура проекта и ключевые компоненты

* **`ContestManagerApplication.java`** — Главный запускаемый класс приложения, активирующий планировщик (`@EnableScheduling`) и конфигурационные свойства.
* **`Telegram/ContestJoinBot.java`** — Основное ядро Telegram-бота. Обрабатывает входящие текстовые сообщения, команды, а также нажатия `Inline`-кнопок (`CallbackQuery`).
* **`Service/SubsCheckService.java`** — Шедулер автоматической проверки подписок. Раз в 45 секунд проверяет всех пользователей со статусом `isParticipates = true`.
* **`Service/JoinRequestService.java` & `listener/JoinRequestListener.java`** — Сервис и слушатель, отвечающие за перехват и отложенное (30 секунд) одобрение заявок на вступление в каналы.
* **`Service/VkService.java` & `Service/TgService.java`** — Сервисы интеграции со сторонними платформами. Выполняют непосредственные запросы к API VK и Telegram.
* **`Service/ContestService.java`** — Бизнес-логика управления состоянием участников: добавление, обновление привязки VK, проверка наличия записей и изменение флагов участия.
* **`Models/ContestUser.java`** — JPA-сущность, описывающая таблицу `contest_users` (содержит Telegram ID, Username, VK ID, флаг участия и временную метку присоединения).
